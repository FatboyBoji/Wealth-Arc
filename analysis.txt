Optimized Session Termination Flow

1. Frontend (When user clicks terminate):
   - Start session termination
   - Immediately initiate new login attempt
   - Show loading state
   - Don't wait for termination to complete

2. Backend Changes:
   a. Add new endpoint /auth/pre-login:
      - Accept new session details
      - Allow temporary excess session (4th)
      - Mark old session for deletion
      - Return success with new session token

   b. Modify session cleanup job:
      - Add priority cleanup for marked sessions
      - Run more frequently for marked sessions
      - Clean within 5 seconds

3. Database Changes:
   ✓ Add is_marked_for_deletion column to sessions table
   ✓ Add marked_at timestamp column
   ✓ Add index for faster cleanup queries

4. Implementation Order:
   1. ✓ Database schema update

      ✓ Added session marking columns

      ✓ Updated TokenResponse type

   3. Session cleanup modification

5. Expected Benefits:
   - Faster user experience
